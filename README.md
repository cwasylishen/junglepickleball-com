# junglepickleball.com

One-page site for Jungle Pickleball, the indoor pickleball destination in Ojochal, Costa Rica. Four indoor courts, free WiFi, and a community hub.

## Stack

Static HTML with a Tailwind CSS build step. Alpine.js loads from CDN. Deployed as a Cloudflare Worker with static assets (`wrangler deploy`), matching the other wizardweb sites.

- `index.html` - the full single-page site (hero, features, gallery, about, events, FAQ)
- `404.html` - branded not-found page
- `src/input.css` - Tailwind entry plus custom styles
- `assets/styles.css` - compiled, minified Tailwind output (generated by the build)
- `assets/hero/`, `assets/features/`, `assets/gallery/` - image sets
- `admin/index.html` - password-protected events admin (add / edit / archive)
- `functions/` - Cloudflare Pages Functions: the events API and admin auth
- `tailwind.config.js`, `package.json` - build config
- `_headers` / `_redirects` - Cloudflare Pages config
- `sitemap.xml`, `robots.txt`

## Events admin

> Status: **dormant in the current static-only deploy.** The site ships as a
> static-assets Worker, so the `functions/` API does not run yet. The home page
> events come from the built-in fallback in `index.html`. To turn the admin on,
> add a `main` Worker entry that routes `/api/*` and serves assets, uncomment the
> KV binding in `wrangler.toml`, and set the env vars below.

Events on the home page are managed at **`/admin`** and stored in Cloudflare KV. The
public page reads them from `GET /api/events`, with a built-in fallback so it still
shows events if the API is ever unreachable. The admin auto-seeds the current events
the first time it loads.

API (Pages Functions in `functions/`):

- `GET /api/events` - public, active events only
- `POST /api/login` / `POST /api/logout` / `GET /api/session` - auth (signed cookie)
- `GET /api/config` - public Turnstile site key for the login page
- `GET|POST /api/admin/events`, `PUT|DELETE /api/admin/events/:id` - protected CRUD

Security: login is gated by Cloudflare Turnstile and per-IP rate limiting, the session
is an HMAC-signed httpOnly cookie (8 hour expiry), and all `/api/admin/*` routes are
guarded by `functions/api/admin/_middleware.js`.

### Required Cloudflare setup

1. **KV binding** - already in `wrangler.toml` (`EVENTS` -> `junglepickleball-events`).
2. **Turnstile** - in the Cloudflare dashboard, create a Turnstile widget for
   `junglepickleball.com` (and `*.pages.dev` for preview testing). Note the site key
   and secret key.
3. **Pages environment variables / secrets** (Pages project -> Settings -> Variables):
   - `ADMIN_USERNAME` (plaintext var) - Roger's username
   - `ADMIN_PASSWORD` (encrypted secret) - Roger's password
   - `SESSION_SECRET` (encrypted secret) - a long random string for signing sessions
   - `TURNSTILE_SECRET_KEY` (encrypted secret) - from the Turnstile widget
   - `TURNSTILE_SITE_KEY` (plaintext var) - from the Turnstile widget

Functions run only on Cloudflare, so the admin cannot be tested with the local Python
server - use the `*.pages.dev` preview after connecting the repo.

## Build

Tailwind scans the HTML and compiles `assets/styles.css`:

```
npm install
npm run build      # one-off build
npm run watch      # rebuild on change during local dev
```

The compiled `assets/styles.css` is committed so the site works without a local build, but Cloudflare regenerates it on every deploy. If you add new Tailwind classes locally, run `npm run build` before previewing.

## Deploy

Deployed via Cloudflare Workers Builds (Git-connected), same as the other sites:

- **Build command:** `npm run build` (compiles `assets/styles.css`)
- **Deploy command:** `npx wrangler deploy`

`wrangler.toml` uses `[assets] directory = "."`, so the whole repo is published as
static assets except the paths listed in `.assetsignore` (source, config, and the
dormant `functions/`). Every push to `main` rebuilds and redeploys.

## Local preview

npm run build
python -m http.server 8080
